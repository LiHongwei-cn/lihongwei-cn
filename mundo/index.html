1 蒙多五步法

定位瓶颈

不知道怎么做？做不好？不够好？先精准定位问题本质。

多渠道并行搜索

同时向多个 AI 模型请教 + 搜索 GitHub Issues / Stack Overflow / 掘金。不依赖单一来源。

渠道搜索方式适用场景
GitHubgithub.com/search?q=...&type=code找开源实现、issue 讨论
Stack Overflowweb_search + site:stackoverflow.com具体技术问题、报错排查
掘金web_search + site:juejin.cn中文技术文章、实战教程
其他 AI 模型通过 API 或 web 端交叉验证多角度方案对比、查漏补缺

四维对比

对收集到的 2-3 种方案，按四个维度打分对比：

维度权重说明
简洁度高代码越少越好 · 依赖越少越好 · KISS 原则
性能中执行效率 · 内存占用 · 可扩展性
可维护性高是否容易理解 · 是否符合项目规范 · 有没有隐藏副作用
匹配度最高是否适配当前技术栈 · R2016b 兼容？跨平台？

吸收实施

理解 → 适配 → 代码落实。绝不直接复制粘贴未理解的代码。适配到当前项目的技术栈和规范。

效果验证 + 沉淀

解决了吗？有副作用吗？有价值的方案用 skill_manage(action='create') 沉淀为可复用的 Skill。

2 并行审计模式

当需要对整个项目进行全面审查时，蒙多可以同时分身多个子代理并行工作。

侦察（1 个代理扫描全局）

用一个代理扫描所有目标文件，输出结构化问题清单（按严重程度：高 / 中 / 低）。

委托一个代理读取所有 index.html，返回问题清单表格

并行修复（3 个代理同时开工）

按问题类别分成 3 个并行任务，各改各的，互不冲突：

优先级任务示例
高404 链接 / 缺失入口 / 下载失效修复死链、补充项目卡片
中页面太单薄 / 缺少教程重写内容、添加 FAQ
低章节编号 / 文本优化统一风格、修正措辞

统一提交

所有修改完成后一次性 git add -A && git commit && git push。

并行任务必须修改不重叠的文件，否则会产生冲突。先侦察后修复，不要边侦察边修。

3 使用方式

🖥️

Claude Code

在 ~/.claude/skills/mundo/ 安装蒙多 Skill，遇到瓶颈自动触发，无需手动调用

⚕️

Hermes Agent

在 ~/.hermes/skills/mundo/ 安装蒙多 Skill，支持 skill_view 按需加载

📥

安装方式

从 Skills 市场 一键下载，复制到对应 skills 目录即可

🌐

消息触发

在对话中说"蒙多""卡住了""搞不定""没思路"等关键词，AI 自动激活蒙多模式

◆ 蒙多铁律

🚫

绝不直接复制粘贴

理解原理后再适配，不搬运未理解的代码

⏱️

单一来源 ≤ 10 分钟

一个来源上花超过 10 分钟还没结果 → 立刻换渠道

🔀

至少对比 2-3 种方案

不要看到第一种可行方案就停下，多看一眼总有更好的

📝

有价值的沉淀为 Skill

解决一次性的问题不值得记住，解决一类问题的方法值得
