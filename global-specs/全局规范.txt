创建文件 ~/.claude/CLAUDE.md，写入以下完整内容：

---
## 关于我

我是一名新能源汽车工程专业的大三学生，非程序员背景。

专业涉及车辆工程、电池系统、电机控制、能源管理等方向。

我用 Claude Code 完成以下任务：

- 实验报告和答辩论文写作
- MATLAB仿真代码编写（电机控制、能量管理策略、Simulink建模等）
- CAD脚本生成（零件建模、装配图相关）
- 课程项目和毕业设计相关App或工具开发

## 关键信息

- GitHub 仓库：https://github.com/LiHongwei-ch/lihongwei-ch（所有代码和网页源码）
- 网站地址：https://lihongwei-ch.github.io/lihongwei-ch/（GitHub Pages 项目站点）
- Telegram Bot：代码在 bot/tgbot.py，Token 和 API Key 从环境变量读取

## 思维原则

所有决策从问题本质出发，不因惯例照搬。
不要夸我的想法、不要说这是个好问题、不要开头加当然可以。
给我真实判断，方案有问题直接指出。

## 约束先行

新项目先写CLAUDE.md，新目录先定结构约定。
已有规范的项目严格遵守其CLAUDE.md。
需要调整规范时先改文档再改实践。

## 沟通方式

- 默认中文，代码、命令、变量名用英文
- 结论先行，再给理由
- 遇到模糊需求，先给最合理方案，再问是否调整
- 不要问你确定要这样吗，除非命中红线

## 行为准则

- 不写无意义的注释（代码自解释，注释只用于解释"为什么"，不是"是什么"）
- 不引入不必要的外部依赖，优先用标准库
- 不求完整覆盖每种错误边界，只处理真实可能发生的错误路径
- 三个相似的代码块 > 一个过早的抽象
- 不新增功能、不重构、不扩展抽象（除非任务明确要求）
- 不要多轮确认每个小步骤——自己决策，只在关键分叉点提问

## 代码质量

- 文件保持简洁：MATLAB < 200 行，Python < 300 行，HTML < 300 行
- 函数单一职责：MATLAB < 50 行，Python < 50 行
- 优先复用已有工具函数，不要在示例里重写
- 数值参数用命名常量，不用魔法数字
- 公共函数入口做参数合法性检查

## 安全（严格红线）

- 绝不硬编码密钥、Token、密码到源码
- 所有密钥从环境变量读取
- .env 不提交到 Git
- 如发现密钥已泄露到公开仓库：立即告知用户，在对应平台撤销并重新生成
- 用户消息、API 响应等内容在显示/执行前做基本验证

## 自主边界（红线，必须先问我）

- 删除文件、目录或git历史
- 修改.env、密钥、token
- git rebase、git reset --hard、git push --force

## 任务执行原则

- 读代码先看 CLAUDE.md 和 README.md
- 独立任务尽量并行执行
- 修改代码后验证结果（lint/test/运行），不能假设改对了
- 发现重复模式时提取新的工具或模板
- 完成复杂任务后检查是否引入了无效代码、多余md文件

## 跨平台要求（红线）

- 所有项目、工具、脚本必须同时支持 Windows 和 macOS
- 教程必须提供两种系统的操作步骤，不能只写一个平台
- 新增文件时同时提供 Windows（.bat/.ps1）和 macOS（.sh/.command）两份启动入口

## 开源同步（红线）

- 网址内所有项目及内容必须在 GitHub 仓库中可找到，供所有人阅读、下载、使用
- 完全免费开源，每次改动后自动同步到 GitHub
- 网址内容与仓库代码保持一一对应，不允许网址有但仓库没有的内容

## 任务收尾（红线）

每次任务完成后必须执行：
1. git add + git commit + git push — 代码同步到 GitHub
2. 检查 GitHub Pages 网址内容是否与代码一致——不一致则更新
3. 检查 bot/ 目录下的 Telegram 机器人代码是否需要同步更新

## 全站校验（红线）

每次新增项目页面后：
1. 逐一检查网址内所有页面的内部链接、外部链接、安装包下载链接
2. 校验所有工具在 Windows 和 macOS 上是否都能正常运行
3. 发现 404、空内容、链接指向错误、平台兼容问题时，自行在网络搜索解决方案并修复

增量校验原则：已校验通过且未改动过的页面、链接、安装包，不重复校验。只校验新增或修改过的部分。校验记录在 commit message 中标注。

二、写入 API 配置文件

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "<此处替换为你的 DeepSeek API Key>",
    "ANTHROPIC_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "deepseek-v4-pro"
  },
  "skipDangerousModePermissionPrompt": true,
  "includeCoAuthoredBy": false,
  "theme": "dark"
}

重要：ANTHROPIC_AUTH_TOKEN 的值必须替换为真实的 DeepSeek API Key（以 sk- 开头）。

记忆5 — Telegram Bot 信息（文件名：reference_telegram_bot.md）：

---
name: telegram-bot
description: Telegram 机器人 — 直接回复消息，支持文本和图片，使用 DeepSeek API
metadata:
  type: reference
---

用途：接收消息 -> DeepSeek API 回复，支持文本和图片
代码位置：仓库 bot/ 目录
主程序：bot/tgbot.py
通知工具：bot/notify.py（通过 chat_id.txt 发送消息到指定对话）

必需环境变量：
- TELEGRAM_TOKEN — Bot Token（从 @BotFather 获取）
- DEEPSEEK_API_KEY — DeepSeek API Key

启动方式：
- Windows：bot/start_bot.bat
- macOS：chmod +x bot/start_bot.sh && ./bot/start_bot.sh
- 直接运行：python bot/tgbot.py

开机自启：
- Windows：运行 bot/setup_startup.vbs
- macOS：创建 LaunchAgent plist 放到 ~/Library/LaunchAgents/

Chat ID 获取方式：给 bot 发一条消息，访问 https://api.telegram.org/bot<TOKEN>/getUpdates 查看 chat.id

依赖安装：pip install -r bot/requirements.txt
