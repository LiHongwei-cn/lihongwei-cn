全局规范部署包

一套规范，部署到所有 AI Agent。
Claude Code / Hermes / Codex / Cursor / Windsurf / Copilot — 一条命令全搞定。

💡 这是什么？

全局规范部署包是一套跨 Agent 的通用行为规范。安装后，你所有正在使用的 AI Agent 都会自动遵循同一套代码风格、安全红线、辩证思维规范。

它支持 6 个 Agent：Claude Code、Hermes Agent、OpenAI Codex、Cursor、Windsurf、GitHub Copilot。安装脚本自动检测你装了哪些 Agent，只部署对应的配置。

一句话：一套规范管所有 AI。改一次，全部生效。

2 安装步骤

🍎 macOS / Linux
🪟 Windows

进入 global-specs/ 目录，然后运行安装脚本：

1

赋予执行权限

给安装脚本添加可执行权限，让脚本可以运行。

chmod +x install.sh
⚠ 常见问题：如果提示 Permission denied，说明文件权限不正确，执行上面的命令即可。
2

运行安装脚本

脚本自动检测已安装的 Agent（Claude Code / Hermes / Codex 等），只部署对应的配置文件到各 Agent 的配置目录。

./install.sh
⚠ 脚本会检测 ~/.claude/、~/.hermes/、~/.codex/ 等目录。未安装的 Agent 自动跳过。
3

编辑 API Key

安装完成后，打开 ~/.claude/settings.json，将占位 Key 替换为你的真实 DeepSeek API Key。

vim ~/.claude/settings.json # 或 open ~/.claude/settings.json
⚠ 常见问题：如果已有 settings.json，安装脚本不会覆盖它。此时需手动对比合并，或删除旧文件重新安装。
4

验证安装

启动 Claude Code，输入 /neat 测试 Skill 是否正常加载。

claude /neat
✓ 如果 Skill 开始盘点文件，说明部署成功。

进入 global-specs\ 目录，然后：

1

双击运行安装脚本

双击 install.bat，或在终端中运行它。脚本会自动检查 Claude Code 是否已安装，然后部署所有文件到 %USERPROFILE%\.claude\。

install.bat
⚠ 常见问题：如果窗口一闪而过，在终端里运行 install.bat 可以看到报错信息。也可以右键 →「以管理员身份运行」试试。
2

编辑 API Key

安装完成后，用记事本打开 %USERPROFILE%\.claude\settings.json，将占位 Key 替换为真实 DeepSeek API Key。

notepad %USERPROFILE%\.claude\settings.json
⚠ 常见问题：如果已有 settings.json，脚本不会覆盖。需手动合并或备份后重装。
3

验证安装

启动 Claude Code，输入 /neat 测试。

claude /neat
✓ Skill 开始盘点文件 → 部署成功。
💡 安装脚本不会覆盖已有的 settings.json 和 settings.local.json，保护你的现有配置。

5 配置 API Key

1

获取 DeepSeek API Key

访问 platform.deepseek.com/api_keys，登录后创建 API Key。新用户注册通常有免费额度。

2

填入配置文件

编辑 ~/.claude/settings.json，将 <你的 DeepSeek API Key> 替换为真实 Key：

{ "env": { "ANTHROPIC_AUTH_TOKEN": "sk-你的真实Key", ... } }
💡 也可以通过环境变量设置：export ANTHROPIC_AUTH_TOKEN="sk-xxx"（macOS/Linux）或 setx ANTHROPIC_AUTH_TOKEN "sk-xxx"（Windows）。

6 常见问题 FAQ

▶ 安装失败怎么办？

按以下顺序排查：

1. 确认 Claude Code 已安装：终端运行 claude --version

2. 确认在 global-specs/ 目录下运行安装脚本

3. macOS/Linux 确认已执行 chmod +x install.sh

4. Windows 用管理员身份运行终端再试

5. 如果还是不行，可以手动复制：将 CLAUDE.md → ~/.claude/CLAUDE.md，rules/*.md → ~/.claude/rules/，skills/* → ~/.claude/skills/

▶ API Key 在哪获取？

本部署包默认使用 DeepSeek 作为模型提供商（通过 Anthropic 兼容接口）。

获取地址：platform.deepseek.com/api_keys

新用户注册后有免费额度。获取后填入 ~/.claude/settings.json 的 ANTHROPIC_AUTH_TOKEN 字段。

如果使用官方 Anthropic API，修改 ANTHROPIC_BASE_URL 为默认值并填入 Anthropic Key 即可。

▶ Windows 和 Mac 有什么区别？

核心功能完全一致。差异仅在安装脚本和路径格式：

• Mac/Linux 用 install.sh，配置路径 ~/.claude/

• Windows 用 install.bat，配置路径 %USERPROFILE%\.claude\

• Windows 上 Claude Code 通过 npm 安装：npm install -g @anthropic-ai/claude-code

• Mac 上可以通过 Homebrew：brew install claude-code

▶ 已有的 settings.json 会被覆盖吗？

不会。安装脚本会自动检测 settings.json 和 settings.local.json 是否已存在：

• 不存在 → 新建并复制模板

• 已存在 → 跳过，保护你的已有配置

如果需要强制覆盖，先删除旧文件再运行安装脚本。

▶ Skills 不自动触发怎么办？

确认以下几点：

1. ~/.claude/skills/ 目录下有对应的 Skill 文件夹

2. Skill 文件夹内有合法的 SKILL.md（含 frontmatter 的 name 和 description）

3. 重启 Claude Code 再试

4. 手动测试：输入 /skill-name 看是否能加载

如果手动也加载不了，检查 SKILL.md 的 YAML frontmatter 格式是否正确。

▶ 如何更新已安装的规范？

重新拉取最新代码后再次运行安装脚本即可。脚本会覆盖 CLAUDE.md、rules/ 和 skills/（但不覆盖 settings.json）。

cd global-specs/ git pull ./install.sh # 或 install.bat (Windows)
▶ 可以只安装部分内容吗？

可以。不需要手动运行完整脚本：

• 只要全局 CLAUDE.md → cp CLAUDE.md ~/.claude/CLAUDE.md

• 只要某几个 Skill → cp -r skills/code-tidy ~/.claude/skills/

• 只要规范文件 → cp rules/*.md ~/.claude/rules/

所有组件相互独立，按需取用即可。
