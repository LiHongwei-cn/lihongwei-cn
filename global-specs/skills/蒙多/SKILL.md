## ⚡ v16 核心优化（稳定可控 · 低 Token · 脚本优先 · 严控轮次）

### 四条铁律

| 原则 | 说明 |
|------|------|
| **稳定可控** | 每步有明确输入输出，不猜测，不跳跃，不凭直觉行动 |
| **低 Token** | 按需加载索引，精简上下文冗余信息，能一句话说清的不写十句 |
| **脚本优先** | 能用脚本验证的不靠推理，减少幻觉，提升稳定性 |
| **严控轮次** | 最多 5 轮推理，轮次越多出错概率越高，能 3 轮解决的不用 5 轮 |

### 工作流程（最多 5 轮）

```
第 1 轮：定位 — 明确问题是什么、卡在哪里。不模糊就直接回答，不启动蒙多流程。
第 2 轮：搜索 — 并行查 3 个来源（web_search + skills_list + 其他 AI）。不无限搜索。
第 3 轮：对比 — 2-3 种方案，四维打分（简洁/性能/可维护/匹配）。不纠结。
第 4 轮：实施 — 选最优方案，脚本验证，代码落地。实施前必须验证可行性。
第 5 轮：沉淀 — 有价值则 skill_manage(action='create')，无价值则跳过。只保存核心逻辑。
```

### 脚本优先规则

- 能用 `terminal()` 跑命令验证的，不靠推理猜测
- 能用 `read_file()` 看代码的，不靠记忆描述
- 能用 `execute_code()` 测试逻辑的，不靠理论分析
- 搜索最多 3 个来源，不无限搜索
- 来源优先级：本地 Skill > 官方文档 > Stack Overflow > GitHub Issues > 博客

### 不触发条件

问题明确、有现成方案、只需简单查询 → 直接回答，不启动蒙多流程。
