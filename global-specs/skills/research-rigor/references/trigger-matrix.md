# 辩证思维触发矩阵

蒙多在收到任务时，按以下矩阵判断是否激活辩证思维模式：

## 强制激活（完整 5 阶段工作流）

| 任务类型 | 触发信号 | 工作流深度 |
|---------|---------|-----------|
| 论文写作 | 论文/实验/研究/假设/验证/方法论 | 完整 5 阶段 |
| 实验设计 | 实验/对照组/变量/样本量/统计检验 | 完整 5 阶段 |
| 数据分析 | 数据/分析/结果/统计/显著性 | 盲假设 + 证据评估 + 复盘 |

## 部分激活（多维评估 + 对抗验证）

| 任务类型 | 触发信号 | 工作流深度 |
|---------|---------|-----------|
| 方案对比 | 方案/对比/选型/评估/比较 | 多维 Rubric + 跨模型对审 |
| 技术选型 | 技术栈/框架/工具/选型/迁移 | 多维 Rubric + 对抗验证 |
| 架构评审 | 架构/设计/评审/DDD/微服务 | 多维 Rubric + 五维攻击 |
| 根因分析 | 调试/排错/根因/bug/性能 | 对抗验证 + 反事实推理 |

## 不激活（直接执行）

| 任务类型 | 原因 |
|---------|------|
| 查资料/搜索 | 信息检索不需要辩证 |
| 写代码/改配置 | 执行性任务，做完验证即可 |
| 简单问答 | 一句话能答的不需要评估框架 |
| 文件操作 | 机械操作，无需判断 |

## 判断规则

1. 任务涉及"证明"/"论证"/"选择"/"比较"/"评估" → 激活
2. 任务涉及"不确定性"（不知道哪个方案更好） → 激活
3. 任务涉及"多个可行方案" → 激活
4. 任务是确定性的（只有一个正确答案） → 不激活
5. 任务是纯执行性的（做就完了） → 不激活
