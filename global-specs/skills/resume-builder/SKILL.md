# 简历制作 Skill

## 触发条件
用户提到：简历、resume、CV、求职、找工作、投简历、实习简历、应聘

---

## 一、设计铁律

### 1.1 配色系统（只用这一套）

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#f0f2f5` | 浅灰底色，不刺眼 |
| 纸张背景 | `#fff` | 纯白，打印无色差 |
| 主文字 | `#1a1a2e` | 深蓝黑，比纯黑柔和 |
| 正文 | `#444` / `#555` | 深灰，阅读舒适 |
| 辅助文字 | `#888` / `#aaa` | 浅灰，时间/标签 |
| 强调色（选一个） | `#2563eb`（蓝）或 `#b45309`（琥珀） | 链接、分段标记、重点标签 |
| 边框 | `#e5e7eb` / `#e8eaed` | 极细，不抢视觉 |
| 重点项目底色 | `#fefce8` 边框 `#fde68a` | 淡黄，温暖但不花 |
| 普通卡片底色 | `#fafbfc` | 微灰白，与纸张区分 |

**禁止**：渐变背景、光晕、阴影过重（box-shadow 不超过 `0 1px 12px rgba(0,0,0,0.08)`）、多色混搭。

### 1.2 字体系统

```css
font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Text', 'PingFang SC', 'Microsoft YaHei', 'Segoe UI', sans-serif;
```

| 元素 | 字号 | 字重 | 行高 | 字间距 |
|------|------|------|------|--------|
| 姓名 | 24-26px | 800 | - | 2-3px |
| 职位行 | 14px | 500 | - | - |
| 分段标题 | 15px | 700 | - | 0.5px |
| 学校/项目名 | 13-14px | 600-700 | - | - |
| 正文描述 | 12-13px | 400 | 1.6-1.7 | - |
| 标签 | 10.5-11px | 400 | - | - |
| 时间/辅助 | 11px | 400 | - | - |

**禁止**：用 `font-weight: 300`（太细看不清）、字号低于 10px（打印模糊）。

### 1.3 间距系统（8px 基准网格）

| 位置 | 间距 |
|------|------|
| 页面上下 padding | 40px |
| 页面左右 padding | 44px |
| section 之间 | 26px |
| section 标题与内容 | 14px |
| 卡片之间 | 9-10px |
| 卡片内 padding | 12-15px |
| 标签之间 gap | 4-5px |
| 联系方式 gap | 18px |

### 1.4 视觉层次

```
第1层：姓名（最大最粗）
第2层：分段标题（粗 + 底部分割线）
第3层：项目/学校名（中粗）
第4层：正文描述（常规）
第5层：标签/时间（最小最淡）
```

用 **粗细 + 大小 + 颜色深浅** 三层区分，不靠颜色种类区分。

### 1.5 绝对禁止的设计元素

- ❌ 深色主题 / 暗黑模式
- ❌ 头像照片（除非用户明确要求）
- ❌ 渐变背景 / 光晕 / 网格纹理
- ❌ 入场动画 / hover 特效 / transform
- ❌ 外部 CDN 字体 / JS 库 / 图标库
- ❌ 大圆角（超过 8px）
- ❌ 分栏布局（左右各一半的那种花哨简历）
- ❌ 进度条 / 星级评分（自评技能水平，毫无意义）
- ❌ QR 码（除非用户要求）
- ❌ 个人座右铭 / 自我评价段落（浪费空间）

## 二、排版铁律

### 2.1 页面尺寸

- A4 纸：210mm × 297mm
- 内容区：约 180mm × 257mm（减去 padding）
- 屏幕宽度：`max-width: 820px`
- 打印 padding：`12mm 14mm`

### 2.2 信息密度控制

**一页简历的合理字数**：中文 600-900 字（不含标签）
- 头部：30-50 字
- 教育：30-50 字
- 技能：只写标签，不写句子
- 重点项目：150-250 字
- 其他项目：每个 80-120 字，2-4 个
- 证书：20-40 字

**超过一页怎么办**：
- 砍掉不相关的项目
- 缩短项目描述（每条不超过 2 行）
- 合并相似技能分类
- 删掉自我评价/兴趣爱好

### 2.3 对齐规则

- 所有文字左对齐（不居中，除了底部 GitHub 链接）
- 同级元素严格对齐（学校名和时间在同一行两端）
- 标签流式排列（flex-wrap），不强制等宽
- section 标题左侧有 4px 宽的强调色竖条

### 2.4 响应式断点

```css
/* 移动端：640px 以下 */
@media (max-width: 640px) {
  .page { padding: 24px 18px; }
  .header { text-align: center; }
  .contact-row { justify-content: center; }
  .skills-grid { grid-template-columns: 1fr; }
  .featured-grid { grid-template-columns: 1fr; }
  .cert-grid { grid-template-columns: 1fr; }
}
```

### 2.5 打印样式

```css
@media print {
  body { background: #fff; margin: 0; padding: 0; }
  .page { padding: 12mm 14mm; max-width: none; box-shadow: none; }
}
```

打印前自检：
- 背景色是否丢失？（浏览器默认不打印背景色，需要用白底）
- 链接是否可点击？（PDF 中应该可点击）
- 页数是否正确？（不超过 2 页）

## 三、内容铁律

### 3.1 头部（Header）

**必须有**：姓名、目标职位、手机、邮箱、GitHub
**不要有**：头像、地址、生日、性别、民族、政治面貌

格式：
```
姓 名
目标职位 · 专业方向
📞 手机 | ✉️ 邮箱 | 🔗 GitHub
```

### 3.2 教育背景（Education）

**必须有**：学校、专业、学历、起止时间
**不要有**：GPA（除非 3.5+/4.0 或前 10%）、课程列表、奖学金（除非国奖）

格式：
```
学校名称                    起止时间
专业 · 学历
```

### 3.3 核心技能（Skills）

**分类方式**（选 3-5 类，每类 5-8 个标签）：
- 按技术领域：AI Agent / 工程仿真 / 编程语言 / 工具平台
- 按岗位需求：前端 / 后端 / 数据库 / DevOps
- 按熟练程度：不写！（自评「精通/熟练/了解」毫无意义）

**标签写法**：
- ✅ `MATLAB/Simulink` `FastAPI` `Git/GitHub`
- ❌ `精通MATLAB` `熟练使用Python` `了解深度学习`

### 3.4 项目经历（Projects）

**每个项目必须包含**：
1. 项目名称（加粗）
2. 时间段
3. 一句话定位（这个项目是什么）
4. 2-3 行描述（做了什么、用了什么、效果如何）
5. 技术栈标签

**描述写法 — STAR 法则**：

| 要素 | 说明 | 示例 |
|------|------|------|
| Situation | 背景 | 「MATLAB R2016b 兼容的仿真代码库」 |
| Task | 任务 | 「AI 自动生成仿真脚本和 Simulink 模型」 |
| Action | 行动 | 「覆盖车辆动力学、电机 FOC、电池 SOC/EKF 等场景」 |
| Result | 结果 | 「含 FFT/滤波/RMS 工具函数库，10+ 可运行示例」 |

**动词清单**（项目描述开头用）：

| 类别 | 推荐动词 |
|------|---------|
| 设计 | 设计、架构、构建、搭建、规划 |
| 开发 | 开发、实现、编写、封装、集成 |
| 优化 | 优化、重构、提升、降低、加速 |
| 分析 | 分析、诊断、定位、评估、验证 |
| 管理 | 管理、维护、部署、监控、协调 |

**禁止的写法**：
- ❌ 「负责 XXX」→ 改成「设计/开发/优化了 XXX」
- ❌ 「参与 XXX 项目」→ 改成「独立完成 XXX」或「主导 XXX」
- ❌ 「学习了 XXX」→ 简历不是学习报告
- ❌ 「熟练掌握 XXX」→ 不自评，用项目证明

### 3.5 量化技巧

能用数字就不用形容词：

| 差 | 好 |
|----|-----|
| 大幅提升了性能 | 性能提升 40% |
| 处理了大量数据 | 日处理 10 万条记录 |
| 很多用户使用 | 服务 500+ 用户 |
| 多个 AI 模型 | 17+ 个 AI 模型智能调度 |
| 持续迭代 | 经历 21 个版本迭代 |
| 丰富的工具 | 14 个内置工具 |

**没有数据怎么办**：
- 用规模代替：「覆盖 6 个仿真场景」「含 10+ 可运行示例」
- 用版本代替：「v1.4.1，21 次迭代」
- 用范围代替：「支持 macOS + Windows 双平台」

### 3.6 重点项目 vs 普通项目

**重点项目**（最多 1 个）：
- 用带色背景区块（淡黄 `#fefce8`）
- 详细描述架构、技术亮点、数据面板
- 2×2 网格展示模块，底部统计数字
- 这是简历的「杀手锏」，面试必问

**普通项目**（2-4 个）：
- 浅灰卡片，一行标题 + 一行描述 + 标签
- 每个控制在 80-120 字
- 体现技术广度，不重复重点项目的技能

### 3.7 证书资质

**值得写的**：
- 与岗位直接相关的（低压电工证 → 电气/汽车岗位）
- 行业准入类的（驾照 → 需要出差的岗位）
- 高含金量的（不写大学英语四级，写雅思 7.0）

**不值得写的**：
- 大学英语四级/六级（除非很高分）
- 计算机二级（没有技术含量）
- 普通话证书（除非教师岗位）
- 各种「参与奖」「优秀奖」

### 3.8 不要写的段落

- ❌ **自我评价**：「本人性格开朗、吃苦耐劳、团队合作能力强」— 废话
- ❌ **兴趣爱好**：「爱好阅读、旅游、摄影」— 浪费空间
- ❌ **求职意向**：已经在头部写了职位，不需要重复
- ❌ **期望薪资**：面试时谈
- ❌ **照片**：除非岗位明确要求

## 五、工作流程

### 5.1 信息收集

1. 看用户现有简历（如有）
2. 看 GitHub 仓库 / 网站项目
3. 看用户档案（SOUL.md / memory）
4. 列出所有可用素材：项目、技能、证书、经历

### 5.2 内容策划

1. 确认目标岗位（问用户，或从项目推断）
2. 选择重点项目（1 个杀手锏 + 2-4 个辅助）
3. 确定技能分类（3-5 类）
4. 按 STAR 法则写项目描述
5. 量化所有可量化的数据

### 5.3 样式制作

1. 用模板 `references/template.html` 作为起点
2. 填充内容
3. 调整间距确保一页放得下
4. 检查响应式和打印样式

### 5.4 自检交付

按下方清单逐项检查，全部通过才交付。

## 八、常见场景处理

### 场景 1：项目太多放不下
- 只留 1 个重点 + 3 个普通
- 与目标岗位无关的砍掉
- 相似项目合并

### 场景 2：项目太少太空
- 课程大作业也算项目
- 开源贡献也算
- 个人工具/脚本也算
- 重点写技术细节，不写「学习了XX」

### 场景 3：跨专业求职
- 强调可迁移技能（编程、数据分析、项目管理）
- 项目经历突出技术实现，弱化专业背景
- 教育背景放最后

### 场景 4：实习简历
- 不需要太多工作经历
- 重点写项目和技术能力
- 课程相关项目可以写
- 竞赛/开源经历加分
