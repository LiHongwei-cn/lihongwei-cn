# 简历制作 Skill

## 触发条件
用户提到：简历、resume、CV、求职、找工作、投简历、实习简历、应聘

## 铁律（红线 — 绝对遵守）

### 设计铁律
1. **白底黑字**，永远不用深色主题。简历是给 HR 看的，不是给程序员看的
2. **不放头像**，除非用户明确要求。头像占空间且不必要
3. **不炫技**，不用渐变背景、光晕、网格纹理、入场动画。简历的核心是内容，不是特效
4. **打印友好**，必须写 `@media print` 样式，Ctrl+P 直接打印不出问题
5. **一页为主**，内容控制在一页 A4（210mm×297mm），最多两页
6. **纯 CSS**，不引入任何外部框架/字体/JS 库，单文件可直接打开

### 配色铁律
- 背景：`#fff`（页面）/ `#f0f2f5`（浏览器底色）
- 文字：`#1a1a2e`（标题）/ `#555`（正文）/ `#888`（辅助）
- 强调色：只用一个，推荐 `#2563eb`（蓝）或 `#b45309`（琥珀），用于分段标记和链接
- 边框：`#e5e7eb` / `#e8eaed`，极细 1px
- 特殊区块背景：淡黄 `#fefce8`（重点项目）/ 淡灰 `#fafbfc`（普通卡片）

### 内容铁律
1. **联系方式一行搞定**：手机 + 邮箱 + GitHub，不多不少
2. **教育背景简短**：学校 + 专业 + 时间，不写 GPA 除非很高
3. **技能分类清晰**：4-6 个分类，每类 5-8 个标签，不堆砌
4. **项目描述用动词开头**：「设计了...」「实现了...」「覆盖了...」，不用「负责...」
5. **重点项目突出**：用带色背景的独立区块，配架构图/数据面板
6. **证书只写有用的**：和岗位相关的、含金量高的

## HTML 结构模板

```
header          → 姓名 + 职位 + 联系方式（居左，底部分割线）
education       → 学校 + 专业 + 时间（一行）
skills          → 2×2 网格卡片分类
featured项目    → 带色背景独立区块（用户最得意的作品）
other项目       → 浅灰卡片列表
certifications  → 2×N 网格
footer          → GitHub 链接 + 更新日期
```

## CSS 设计规范

### 排版
- 字体栈：`-apple-system, BlinkMacSystemFont, 'SF Pro Text', 'PingFang SC', 'Microsoft YaHei', 'Segoe UI', sans-serif`
- 姓名：24-26px，font-weight 800，letter-spacing 2-3px
- 分段标题：15px，font-weight 700
- 正文：12.5-13px，line-height 1.6-1.7
- 标签：10.5-11px

### 间距
- 页面 padding：40px 上下，44px 左右
- section 间距：26px
- 卡片内 padding：12-14px
- 卡片间距：9-10px

### 圆角
- 大区块：8px
- 小卡片：6px
- 标签：4px
- 不用大圆角（14px+），简历要正式

### 响应式
```css
@media (max-width: 640px) {
  .page { padding: 24px 18px; }
  .header { text-align: center; }
  .skills-grid { grid-template-columns: 1fr; }
  .cert-grid { grid-template-columns: 1fr; }
}
```

### 打印
```css
@media print {
  body { background: #fff; margin: 0; padding: 0; }
  .page { padding: 12mm 14mm; box-shadow: none; }
}
```

## 常见错误（必须避免）

| 错误 | 正确做法 |
|------|---------|
| 深色主题 | 白底黑字 |
| 放头像 | 不放，除非用户要求 |
| 渐变/光晕/动画 | 纯净白底，最多一个淡色背景区块 |
| 外部 CDN 引用 | 纯内联 CSS，单文件 |
| 技能写「精通」 | 写具体技术栈标签，不自评水平 |
| 项目描述写流水账 | 动词开头 + 量化数据 + 技术栈 |
| 联系方式写一大片 | 一行：手机 + 邮箱 + GitHub |
| 字体太大/太小 | 姓名24-26px，正文12-13px |
| 没有打印样式 | 必须写 @media print |
| 没有响应式 | 移动端单列布局 |

## 工作流程

1. **收集信息**：先看用户现有简历/项目/仓库，提取所有可用素材
2. **确认定位**：目标岗位、突出哪些经历、弱化哪些
3. **写内容**：先写文字内容，确认后再做样式
4. **做样式**：按本 Skill 规范制作 HTML
5. **自检**：按下方清单逐项检查
6. **交付**：写到桌面，告知文件路径

## 自检清单（每次交付前必须过）

- [ ] 白底？（不是深色主题）
- [ ] 无头像？（除非用户要求）
- [ ] 无外部依赖？（CDN、字体、JS 库）
- [ ] 打印友好？（@media print）
- [ ] 响应式？（移动端可读）
- [ ] 单文件？（一个 HTML 搞定）
- [ ] 内容准确？（项目名、版本号、时间正确）
- [ ] 无硬编码密钥？
- [ ] 联系方式正确？
- [ ] 页数控制？（1-2 页）

## 技术栈标签颜色规范

给项目加标签时，同类技术用同色，不同类用不同色：
- 蓝色系（`#2563eb`）：编程语言、框架、通用工具
- 绿色系（`#16a34a`）：前端/小程序/客户端
- 琥珀色系（`#b45309`）：AI/Agent/重点项目
- 紫色系（`#7c3aed`）：DevOps/基础设施
