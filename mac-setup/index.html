macOS · 从零部署

MacBook Air
Claude Code 部署指南

从新机到可用：安装 Claude Code → 接入 DeepSeek API → 导入全部全局规范 → 一键启动洁癖.skill。
每一步精确到文件路径，照做即可。

0 前提条件

你需要有 2 个东西

① DeepSeek API Key — 去 platform.deepseek.com/api_keys 创建
② 本仓库的 global-specs/ 文件夹 — 已经躺在 GitHub 上，下一步会 clone

最终你会得到什么

一台装了 Claude Code 的 MacBook，自动记住你的偏好（中文回复、自动 push、洁癖审查），新项目自动继承规范，每次任务收尾运行 /neat 存档。

1 安装 Claude Code

通过 Homebrew 安装 推荐

brew install claude-code

如果没有 Homebrew，先去 brew.sh 装。

备选：npm 全局安装

适用于不用 Homebrew 的情况：

npm install -g @anthropic-ai/claude-code

验证安装

claude --version

输出版本号即成功。此时 ~/.claude/ 目录已自动创建。

2 拉取全局规范文件

Clone 仓库

# 克隆到桌面（或任意你喜欢的位置） cd ~/Desktop git clone https://github.com/LiHongwei-ch/lihongwei-ch.git

进入 global-specs 目录

cd ~/Desktop/lihongwei-ch/global-specs

后续所有操作都从这个目录出发。用 ls 看一下，应该看到 CLAUDE.md、settings.json、rules/、skills/、memory/ 等文件。

3 配置 DeepSeek API 需要改 Key

编辑 settings.json，填入真实 API Key

用任意文本编辑器打开 settings.json，找到这一行：

"ANTHROPIC_AUTH_TOKEN": "<你的 DeepSeek API Key>"

把 <你的 DeepSeek API Key> 替换为你的真实 Key（以 sk- 开头）。

整个文件只需改这一个值，其他配置（Base URL、Model）已经写好了。

复制到 Claude Code 配置目录

cp settings.json ~/.claude/settings.json

文件内容概览：

// ~/.claude/settings.json 最终效果 { "env": { "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic", "ANTHROPIC_AUTH_TOKEN": "sk-xxxxxxxx", ← 你的真实 Key "ANTHROPIC_MODEL": "deepseek-v4-pro", ... }, "skipDangerousModePermissionPrompt": true, "theme": "dark" }

4 部署用户全局指令

复制 CLAUDE.md 到全局位置

cp CLAUDE.md ~/.claude/CLAUDE.md

这个文件告诉 Claude Code：你是谁、做什么的、行为准则、安全红线。所有项目共享，不需要每个项目复制一份。

作用范围：全局生效。你在任何目录打开 Claude Code，它都会先读这个文件。

5 部署项目规范模板（6 份）

复制 rules/ 到全局

mkdir -p ~/.claude/rules cp rules/*.md ~/.claude/rules/

6 份规范文件一览：

~/.claude/rules/ ├── agents.md ← Agent 调度：何时用 planner/code-reviewer/tdd ├── coding-style.md ← 代码风格：不可变数据、KISS、命名规范 ├── git-workflow.md ← Git 规范：commit message 格式、PR 流程 ├── performance.md ← 模型选择：Haiku/Sonnet/Opus 使用策略 ├── security.md ← 安全红线：密钥管理、commit 前检查清单 └── testing.md ← 测试要求：80% 覆盖率、TDD 流程
这些文件是项目级 CLAUDE.md 的"上游引用"——不是每个项目都要抄一遍，而是 Claude Code 自动加载。

6 安装洁癖.skill

复制 skill 文件

mkdir -p ~/.claude/skills/neat-freak/references cp skills/neat-freak/SKILL.md ~/.claude/skills/neat-freak/ cp skills/neat-freak/references/*.md ~/.claude/skills/neat-freak/references/

安装后文件结构：

~/.claude/skills/neat-freak/ ├── SKILL.md ← 主 skill：5 步审查流程 └── references/ ├── sync-matrix.md ← 变更影响矩阵（什么改动→改哪些文件） └── agent-paths.md ← 各 Agent 平台路径速查

触发方式

在 Claude Code 对话中输入以下任意一个即可触发：

/neat 整理一下 同步一下 梳理一下 收尾

Skill 会自动盘点所有 .md 文件 → 用变更影响矩阵识别要改什么 → 按 docs/ → CLAUDE.md → memory 的顺序修改 → 自检 → 输出变更摘要。

7 部署权限白名单

复制 settings.local.json

cp settings.local.json ~/.claude/settings.local.json

这个文件预授权了常用命令（git add/commit/push、curl 等），避免每次都要点"允许"。

注意：这个文件里有些路径是从 Windows 复制过来的（如 C:\Users\HP\...），在 Mac 上不会生效，但不影响使用。后续在 Mac 上操作时，Claude Code 会学习新的权限路径。

8 导入跨会话记忆

记忆文件是什么

memory/ 目录包含 3 条记忆文件：

memory/ ├── MEMORY.md ← 记忆索引 ├── feedback_auto_push.md ← 代码改动后自动 push 到 GitHub ├── feedback_auto_sync.md ← 任务完成后自动 commit+push └── feedback_file_location.md ← 项目文件位置约定

部署方法

记忆文件与项目路径绑定。首次在项目目录启动 Claude Code 后，Claude Code 会自动在 ~/.claude/projects/ 下创建对应目录：

# 1. 先进入项目目录，启动一次 Claude Code（让它自动创建 memory 目录结构） cd ~/Desktop/lihongwei-ch claude # 进入对话后输入 /exit 退出即可 # 2. 找到自动生成的 memory 目录 ls ~/.claude/projects/ # 3. 把记忆文件复制进去 cp ~/Desktop/lihongwei-ch/global-specs/memory/*.md \ ~/.claude/projects/<自动生成的路径>/memory/
实际上最省事的做法：直接在 Claude Code 里说"记住：代码改动后自动 push 到 GitHub"，它会自动写到正确的 memory 目录。这 3 条记忆只是给你参考用的备份。

9 初始化项目 CLAUDE.md

为每个项目复制模板

project-CLAUDE.md 是一个项目级模板（MATLAB/Simulink 规范），适用于新能源汽车仿真项目：

# 对于 li hongwei-ch 这个仓库（已经是你的主项目） cp ~/Desktop/lihongwei-ch/global-specs/project-CLAUDE.md \ ~/Desktop/lihongwei-ch/CLAUDE.md # 对于其他新项目 cp ~/Desktop/lihongwei-ch/global-specs/project-CLAUDE.md \ /你的新项目目录/CLAUDE.md # 然后根据项目实际情况修改

10 验证部署

启动 Claude Code

cd ~/Desktop/lihongwei-ch claude

测试洁癖.skill

在对话中输入：

/neat

如果看到 skill 开始盘点文件、输出审查结果，说明部署成功。

测试 API 连接

随便问一个问题（比如"当前目录有哪些文件"），如果能正常回复，说明 DeepSeek API 连接正常。

📋 文件映射总表

一目了然——每个文件从哪来、到哪去、为什么需要它。

源文件（global-specs/ 内）目标位置（macOS）作用

🔄 日常工作流

开始工作

打开终端，进入项目目录，启动 Claude Code，直接描述问题即可。无需每次给上下文——全局 CLAUDE.md 和项目 CLAUDE.md 已经告诉了 Agent 一切。

收尾存档 关键步骤

每次完成一个功能或修完一个 bug，关闭对话前运行：

/neat

洁癖.skill 会审查项目文档和记忆，确保代码迭代后文档也跟上了。这相当于游戏存档——下次新开对话，Agent 读到的是最新的规范。

坚持每次收尾跑一遍 /neat，你的 Agent 会越用越聪明。过期的删、重复的合、模糊的改。

新项目启动

新项目第一步永远是先写 CLAUDE.md（可以从 project-CLAUDE.md 模板开始），再开始写代码。约束先行。

❓ 常见问题

启动 Claude Code 报 "command not found"

检查安装路径是否在 PATH 中。brew 安装通常自动配置；npm 全局安装的话确认 npm list -g @anthropic-ai/claude-code 能查到。

API 连接失败

检查 ~/.claude/settings.json 中 ANTHROPIC_AUTH_TOKEN 是否填了正确的 DeepSeek Key、ANTHROPIC_BASE_URL 是否为 https://api.deepseek.com/anthropic。

/neat 没反应

确认 ~/.claude/skills/neat-freak/SKILL.md 文件存在且内容完整。也可以尝试输入"整理一下"触发。

记忆文件不存在 / 没生效

记忆文件与项目路径绑定。首次在项目目录启动 Claude Code 后它会自动创建 memory 目录。也可以在对话中直接说"记住：xxxx"，Claude Code 会自己写。
