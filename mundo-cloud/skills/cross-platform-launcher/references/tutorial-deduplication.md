# 教程页面去重与交叉引用

当两个或多个页面覆盖重叠内容时（如"一键启动器"和"完整指南"），用户容易混淆，导致重复操作。

## 问题模式

典型场景：
- `hermes-launcher` 有"前置条件"（安装 Hermes Agent）
- `hermes-guide` 有完整的安装步骤
- 用户两边都看了一遍，导致重复安装

## 解决方案

### 1. 起始页添加"分流提示"

在**快速入门**页面（如 launcher）的重叠章节前添加醒目标注：

```html
<div class="tip info" style="margin-bottom:16px">
  <strong>还没安装？</strong>请先看 <a href="../完整指南/">完整指南</a> 的安装章节，安装完成后再回来。<br>
  <strong>已经安装过了？</strong>跳过下面，直接下载启动器即可。
</div>
```

### 2. 详细教程页添加"快速入口"

在**完整教程**页面（如 guide）的 hero 区或重叠章节前添加：

```html
<div class="disclaimer">
  本教程是从零开始的完整安装指南。如果你已经安装过，只需要桌面快捷启动，
  请直接去 <a href="../启动器/">一键启动器页面</a>。
</div>
```

### 3. 重叠章节改为"验证"而非"重复操作"

原来：launcher 页面重复了安装步骤
优化后：launcher 只放"确认已安装"检查命令

```html
<!-- 之前：重复安装步骤 -->
<h3>安装 Hermes Agent</h3>
<div class="cmd">curl -fsSL ... | bash</div>

<!-- 之后：只验证，不重复 -->
<h3>确认 Hermes Agent 已安装</h3>
<div class="cmd">hermes --version</div>
<p>输出版本号说明已安装。</p>
```

### 4. 详细教程中交叉引用快速入口

在详细教程的相关章节中，指向更简单的替代方案：

```html
<div class="tip green" style="margin-bottom:16px">
  <strong>更简单的方法：</strong><a href="../启动器/">一键启动器页面</a> 
  提供了现成的创建命令，复制粘贴即可完成，无需手动创建文件。
</div>
```

## 原则

1. **不重复内容** — 同一个操作只在一个页面详细描述，其他页面交叉引用
2. **分流明确** — 用户一眼看出"我该看哪个页面"
3. **验证优于重复** — 快速页面只放验证命令，不放完整安装步骤
4. **双向引用** — 快速页指向详细教程，详细教程也指向快速页
