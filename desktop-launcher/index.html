Claude Code 桌面快捷启动

双击桌面图标即可启动 Claude Code 对话框，自动进入工作目录。
无需每次打开终端输命令。

1 工作原理

桌面快捷方式 → powershell.exe → launcher.ps1 → Windows Terminal → Claude Code

快捷方式指向 powershell.exe，由 PowerShell 调用 launcher.ps1 启动 Windows Terminal，
Terminal 自动设置环境变量并运行 Claude Code 进入你的项目目录。

2 一键创建快捷方式

下载仓库文件 仅需 3 个文件

将以下文件下载到你的项目目录 tools/ 文件夹中：

① create-shortcut.ps1 — 创建快捷方式的脚本
② launcher.ps1 — 启动器脚本（快捷方式实际调用的）
③ claude-icon.ico — 桌面图标

也可以直接 clone 整个仓库，tools/ 目录里所有文件都有。

修改工作目录（首次使用必须）

用记事本打开 create-shortcut.ps1 或 launcher.ps1，找到 $workDir 这一行，改成你自己的项目路径：

$workDir = "C:\Users\你的用户名\Desktop\你的项目"

两个脚本都要改，共 2 处。

运行创建脚本 需要 PowerShell

右键 create-shortcut.ps1 → "使用 PowerShell 运行"，桌面会立即出现 Claude Code.lnk 快捷方式。

如果右键菜单没有 PowerShell 选项，在文件所在目录的地址栏输入 powershell 回车，然后运行：

.\create-shortcut.ps1

双击桌面图标启动 完成

双击桌面 Claude Code 图标，Windows Terminal 会自动打开并启动 Claude Code，光标出现在输入框后即可开始对话。

3 备用方案：bat 启动脚本

如果你不想创建桌面快捷方式，也可以直接用 .bat 脚本启动。创建一个文本文件，重命名为 claude-desktop.bat，写入以下内容：

@echo off :: Work directory - auto-detected (script location's parent folder) set WORK_DIR=%~dp0.. :: Force CLI mode, prevent auto-open VS Code set CLAUDE_CODE_ENTRYPOINT=cli cd /d "%WORK_DIR%" :: Prefer Windows Terminal for interactive UI where wt.exe >nul 2>&1 if %errorlevel% equ 0 ( start "" wt.exe -d "%WORK_DIR%" cmd /k "set CLAUDE_CODE_ENTRYPOINT=cli && claude --dangerously-skip-permissions --no-chrome" exit /b ) :: Fallback: launch directly in current console claude --dangerously-skip-permissions --no-chrome

%~dp0.. 会自动取脚本所在目录的上级目录作为工作目录。如果把 bat 放在项目根目录的 tools/ 里，无需手动修改路径。

4 常见问题

⚠ 双击快捷方式没反应

原因： PowerShell 执行策略禁止运行脚本，或 launcher.ps1 中的路径配置错误。

解决：
① 以管理员身份打开 PowerShell，运行 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
② 确认 launcher.ps1 中的 $workDir 指向了正确的目录
③ 重新运行 create-shortcut.ps1

⚠ 双击后弹出黑色旧式窗口，看不到对话界面

原因： 系统没有 Windows Terminal，Claude Code 的交互界面需要 Windows Terminal 支持。

解决： 打开 Microsoft Store 搜索 "Windows Terminal" 安装（免费），然后重新双击快捷方式。

⚠ 启动时自动打开了 VS Code

原因： Claude Code 检测到已安装 VS Code 后尝试 IDE 集成。

解决： 启动命令中已包含 set CLAUDE_CODE_ENTRYPOINT=cli，强制终端模式。如果问题仍然存在，检查你的 Claude Code 全局设置中是否开启了 IDE 集成。

⚠ 一次打开了两个窗口

原因： 旧版启动器使用 -w 0 new-tab 尝试连接已有窗口，在部分环境下行为不稳定。

解决： 确认 launcher.ps1 中的 wt.exe 参数为 -w -1 new-tab（强制创建独立新窗口）。仓库中已是最新版本。

（可选）配置代理

如果网络需要代理才能访问 DeepSeek API，编辑 launcher.ps1，在启动命令前加一行：

$env:HTTPS_PROXY = "http://127.0.0.1:7897"

把端口改成你实际使用的代理端口。

（可选）更换工作目录

如果项目搬家了，只需编辑 launcher.ps1 中的 $workDir 变量，无需重新创建快捷方式。
