# 蒙多超时问题修复报告

**修复时间**: 2026-06-11 06:43:07
**修复版本**: v2.0.9

## 问题分析

### 根本原因
1. **工具调用次数过多** - 一个简单任务调用 21 次工具
2. **LLM 调用延迟** - 每次工具调用后都需要等待 LLM 响应
3. **缺乏超时保护** - 单个工具和任务级别都没有超时限制
4. **工具选择不智能** - 使用了错误的工具完成任务

### 超时场景
- T3 文件处理：search_files 工具执行 85 秒
- T5 复杂任务：调用 21 次工具，总耗时 120 秒

## 修复方案

### 1. 任务级超时保护
```python
TASK_TIMEOUT = 90  # 任务级超时：90秒
```
- 任务执行超过 90 秒自动强制完成
- 基于已有信息生成最终回复

### 2. 工具调用次数限制
```python
MAX_TOOL_CALLS = 10  # 单任务最大工具调用次数
```
- 限制单个任务最多调用 10 次工具
- 防止无限循环调用

### 3. 单个工具超时保护
```python
TOOL_TIMEOUT = 30  # 单个工具超时：30秒
```
- 使用线程池执行工具
- 超时后自动取消并返回错误信息

### 4. 优化工具选择
- 修改工具提示，引导使用正确的工具
- 统计文件数量用 terminal 运行 find/wc
- 搜索文件内容用 search_files
