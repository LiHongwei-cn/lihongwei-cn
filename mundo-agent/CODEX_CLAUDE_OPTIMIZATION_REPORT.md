# 蒙多 Claude Code 优化报告

## 优化背景

根据文章《把 Claude Code 调教成我的赛博打工人》和《48小时极限测试：Claude Code的20个致命痛点》，我们发现：

1. **Token浪费**：Claude Code添加多余文本（解释、总结、思考过程）
2. **缓存机制**：Claude Code使用前缀匹配缓存，系统提示变化会导致缓存失效
3. **上下文过长**：SKILL.md（1535行）每条消息都加载，消耗大量token
4. **任务跑偏**：Claude Code会添加解释性文字，偏离核心任务

## 优化措施

### 1. 优化 claude_integration.py

**新增功能：**

- **上下文压缩**：`_compress_context()`函数
  - 移除多余空白行
  - 移除独立行注释（保留代码内注释）
  - 压缩token消耗

- **智能路由**：`_smart_effort()`函数
  - 根据任务复杂度自动选择努力级别
  - 简单任务（文件操作、查询）→ `low`
  - 中等任务（代码生成、调试）→ `medium`
  - 复杂任务（架构设计、重构）→ `high`

- **重试机制**：`exec_with_retry()`方法
  - 失败时自动重试
  - 最多重试2次

- **输出清理优化**：
  - 移除中英文思考过程
  - 移除中英文任务描述重复
  - 移除中英文文件操作说明

**移除无效选项：**

- 移除`--no-markdown`选项（Claude Code不支持）

### 2. 更新 delegation.py

- 使用`exec_smart()`替代`exec_full_power()`
- 根据任务复杂度自动选择努力级别

### 3. 优化策略

#### 3.1 Token优化

1. **上下文压缩**：自动压缩用户消息，减少token消耗
2. **输出清理**：移除Claude Code添加的多余文本
3. **智能努力级别**：简单任务用低努力级别，减少token消耗

#### 3.2 缓存优化

1. **保持前缀一致性**：系统提示保持稳定
2. **减少SKILL.md变化**：避免频繁修改SKILL.md

#### 3.3 任务优化

1. **精准指令**：明确任务目标，减少Claude Code的自由发挥
2. **输出格式**：使用`--output-format text`，禁用markdown
3. **最小模式**：使用`--bare`模式，跳过不必要的初始化

3. **任务准确性提高**：精准指令和输出清理，减少任务跑偏
